# FrameFlow Navigation Architecture — Verification

This document defines the **canonical navigation architecture** for the FrameFlow application.

Any implementation generated by Claude Code must follow this structure.

If the implementation diverges from this architecture, it should be corrected.

Claude must verify the navigation structure after generating code.

---

# TOP LEVEL MENU

The application must contain exactly these primary sections:

1. Hoje
2. Planeamento
3. Filme
4. Equipa
5. Departamentos
6. Pós
7. Mirror

No additional top-level sections should be created.

---

# LOCATIONS NAMING RULE

Locations must exist in **three distinct contexts**:

1. Locations (Department)
2. Locais da História (Story Locations)
3. Locais de Filmagem (Film Locations)
4. Pontos Logísticos (Logistics Points)

These must not be merged into a single concept.
